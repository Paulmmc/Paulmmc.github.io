知识点十一 正则表达式

正则表达式：正则表达式使用单个字符串描述、匹配一系列符合某个句法规则的字符串。在很多文本编辑器里，正则表达式通常被用来检索、替换那些符合某个模板的文本。
原义文本字符：正常的文本字符
元字符：具有特殊意义的专用字符，是代替正常文本字符的字符
限定符：限定匹配的数量或特殊条件

元字符：
.   匹配除换行以外的任意字符
\w  匹配字母或数字或下划线或汉字
\s  匹配任意的空白符
\d  匹配数字
\b  匹配单词的开始或结束
^   匹配字符串的开始
$   匹配字符串的结束
[a-z]       匹配小写a到小写z
[a-zA-Z0-9] 匹配小写a到小写z，大写A到大写Z，0到9

限定符：
{n}     重复n次
{n,}    重复大于等于n次
{n,m}   重复n次到m次
*       重复大于等于0次
+       重复大于等于1次
?       重复0次或1次

例子：
全数字      ^[0-9]$
电话号      \d{3}-\d{8} | \d{4}-\d{7}
QQ号        [1-9][0-9]{4,}
账号(字母开头，5-16位，允许字母数字下划线)
^[a-zA-Z][a-zA-Z0-9_]{4,15}$        

//正则表达式需要放在 / 这里 /g，也就是 / 和 /g 之间
let reg = / ^[0-9]$ /g
let str = "15354ww";
//str是否匹配
let res = reg.exec(str);
//打印1，表示有一个匹配的，不匹配则不打印
document.write(res.length + "");
//匹配的内容
res.forEach((value,index)=>{
    //value表示具体的匹配字符
    //index表示索引
    document.write(value + "" + index);
});

三、观察者模式

我

监听人
监听人
监听人

若我状态改变
所有监听人都可以得到消息

interface IObserver
{
    nameChanged(newName:string);
}

class Person
{
    private _name:string;
    //所有观察者
    observers: Array<IObserver> = new Array<IObserver>();

    set name(value)
    {
        this._name = value;
        for(let observer of this.ovservers)
        {
            observer.nameChanged(this._name);
        }
    }

    get name()
    {
        this._name = value;
    }


}

class Test implements IObserver
{
    nameChanged(newName:string)
    {
        document.write("监听变化，名字变为" + newName);
    }
}
