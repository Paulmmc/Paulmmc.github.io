知识点一 C++介绍

C++具有面向对象的特性
与C#和JAVA不同，C#和JAVA会在被编译为一个中间代码，然后再程序运行时，虚拟机再翻译为机器码
C++可以被编译器直接编译为机器码，机器码直接控制硬件
因此，C++几乎可以在任何平台上使用

知识点二 C++是如何工作的

#include <iostream>

int main()
{
    std::cout << "Hello World!" << std::endl;
    std::cin.get();
}

一、预处理

在以上实例代码中，
第一行，#include
所有在#以后的语句都是预处理语句，编译器在收到源文件后，先处理这个预处理语句
#include 语句的意思是: 他会找到一个iostream 文件，并且拷贝进来使用，这些文件一般称为“头文件”

二、main函数

是程序的入口
程序运行时，从这个函数开始执行代码
main函数比较特殊，不需要返回int，若不写返回值，默认返回0

三、C++转化为可运行的二进制文件流程

  1. 编译器对预处理语句进行预处理
  2. 项目中的每一个cpp文件都会被单独编译为一个.obj类型文件，而头文件不会被编译
  3. 链接器将所有obj文件合并成一个.exe文件

知识点三 C++编译器是如何工作的

一、简介

一段C++代码是如何转变为可执行的二进制文件的？
先被编译，再被链接
编译器的作用就是，将一段代码编译成一个成为目标文件的中间格式，例如将.cpp文件编译为.obj文件

二、编译器是如何将.cpp编译为.obj的

  1. 先将预处理语句进行预处理(最常用的预处理语句是 #include 是将头文件的代码直接复制粘贴过来)
  2. 编译器将.cpp文件编译为.obj文件(机器码)

知识点四 C++链接器是如何工作的

一、简介

编译器将每个脚本编译为一个个单独的.obj格式的二进制文件后，链接器将他们链接到一起，生成一个.exe的可执行文件
链接器会是找到每个符号和函数，将他们链接在一起

二、inline关键字

inline void Log(const char* message)
{
    std::cout << message << std::endl;
}

//当使用 具有inline关键字的函数时，以下InitLog函数，等同于InitLog1函数
void InitLog()
{
    Log("InitLog");    
}

void InitLog1()
{
    std::cout << "InitLog" << std::endl;
}

三、注意

声明可以重复，但定义只能有一个

//这是声明，可以重复
void Log(const char* message);
void Log(const char* message);

//这是定义，只能有一个
void Log(const char* message)
{
    std::cout << message << std::endl;
}

二、基本结构

int Fun(int a, int b)
{
    retrun a + b;
}

三、注意

过多的函数会影响
代码的可读性
程序的运行效率

四、声明和定义

//这是声明，可以重复
void Log(const char* message);
void Log(const char* message);

//这是定义，只能有一个
void Log(const char* message)
{
    std::cout << message << std::endl;
}

因此声明一般会写在头文件中，告诉这个脚本，有这个函数
而定义会写在脚本中，若写在头文件中，容易重复定义，造成链接错误

一、简介

二、#pragma once 语句

在创建一个头文件时，VS会自动在头文件脚本中为我们加上 #pragma once
#pragma once 可以阻止我们单个头文件被多次包含

知识点九 条件与分支

一、简介

当条件符合时，会执行特定语句块
本质上，在程序运行时，程序会加载到内存中，当有条件分支语句时，它会在内存之间跳跃，这表示if语句会有比较大的开销

二、例子

int main()
{
    int x = 5;
    if(x == 5)
    {
        std::cout << "Hello World" << std::endl;
    }

    const char* ptr = nullptr;
    if(ptr)
        std::cout << ptr << std::endl;
    else if(ptr == "h")
        std::cout << "h" << std::endl;
    else
         std::cout << "ptr is null!" << std::endl;
}

知识点十二 C++指针

一、简介

指针是int，一种存储内存地址的数字

二、声明一个指针

int main()
{
    //一个变量类型后面加 * ，表示这个类型是一个地址类型，而前面的变量类型是无意义的，只是给我们看的
    //为地址赋值为0，0不是一个有效的内存地址，说明这个是无效指针
    void* ptr = 0;
    //与0同理
    ptr = NULL;
    //与0同理
    ptr = nullptr;

   
}

三、获得一个变量的地址

int main()
{
    int var = 8;
    //在一个值前面加 & ，表示他的地址，此时将 var 变量的地址赋给了 intptr
    void* voidptr = &var;
    int* intptr1 = &var;
}

四、逆向指针

int main()
{
    int var = 8;
    void* voidptr = &var;
    int* intptr1 = &var;

    //逆向指针
    //可以通过地址，为地址指向的内存赋值
    //会报错，因为前面是 void* ，编译器不知道他的内存大小
    voidptr = 10;
    //不会报错，因为声明时，指定了类型 int*，编译器知道他的内存大小为4个字节
    *intptr1 = 10;
}

五、指定电脑分配指定大小的内存

int main()
{
    //指定电脑分配指定大小的内存
    //一个char是一字节，八个char的数组，就是八字节，此时我们让电脑分配了一个八个字节的内存
    char* buffer = new char[8];
    //将buffer指向的地址，前8字节(这里buffer刚好是八个字节，所以全部都被设置为0)，设置为0，用于初始化内存
    memset(buffer, 0, 8);
    //删除 buffer 数据，释放内存
    delete[] buffer;
}

六、二级指针

int main()

    int a = 5;
    int* intPtr = &a;

     //指向指针的指针
    int** ptr = &intPtr;

知识点十三 c++引用

一、简介

引用只是指针的伪装，作用与指针相似，是指针的语法糖
引用不占空间，引用不能像二级指针一样，引用一个引用
声明一个引用时，必须对他赋值

二、写法

//普通
void Increment(int value)
{
    value++;
}

//指针
void IncrementPtr(int* value)
{
    (*value)++;
}

void IncrementRef(int& value)
{
    value++;
}


int main()
{
    int a = 5;
    //一个引用，不是变量，引用了a
    int& ref = a;
    //改变该引用，同时改变了a本身，类似别名
    ref = 2;

    //普通增加，函数内部复制了一个value变量，对这个复制体进行了增加，对外部的 a 没有影响
    Increment(a);

    //使用了指针，传递的是 a 这个变量的地址，内部对其内存进行了更改，所以外部的 a 也会更改
    IncrementPtr(&a);

    //使用了引用，只是引用了a，内部会对外部的 a 进行增加
    IncrementRef(a);
}

三、注意

int main()
{
    int a = 5;
    int b = 8;

    //声明一个引用时，必须对他赋值
    int& ref = a;
    //此时并不是改变了引用，引用变为了引用b，而是直接将 a 赋值成了 b
    ref = b;
}

知识点十四 c++类

一、简介

封装、继承、多态
数据和功能组合在一起

二、基本使用

class Player
{
    //类的成员，默认为private
public:
    int x, y;

    {
        x += x * speed;
        y += y * speed;
    }
};

{
    Player player;

三、结构体

1.使用

struct position
{
    int x;
private:
    int y;
};

int main()
{

}

2.与class区别

结构体成员默认为 public，与class不同
struct结构体存在的原因是，希望与C语言保持向后兼容，因为C语言没有类，只有结构体

3.何时使用

结构体和类只存在一点区别，何时使用因人而异
一般来说
结构体用于表示一堆变量集合，比如Position、Vector2，里面的方法都是在操纵这些变量
结构体一般不会用继承
类用于表示很多功能集合，比如一个玩家类Player

三、如何写一个类

//比如写一个Log类
class Log
{
public:
    const int LogLevelError = 0;
    const int LogLevelWarning = 1;
    const int LogLevelInfo = 2;
private:
    int m_LogLevel = LogLevelInfo;
public:
    void SetLevel(int level)
    {
        m_LogLevel = level;
    }

    void Error(const char* message)
    {
        if(m_LogLevel >= LogLevelError)
            std::cout << "[ERROR]: " << message << std::endl;
    }
    
    void Warn(const char* message)
    {
        if(m_LogLevel >= LogLevelWarning)
            std::cout << "[WARNING]: " << message << std::endl;
    }

    void Info(const char* message)
    {
        if(m_LogLevel >= LogLevelInfo)
            std::cout << "[INFO]: " << message << std::endl;
    }
};

int main()
{
    Log log;
    log.SetLevel(log.LogLevelWarning);
    log.Warn("Hello");

    std::cin.get();
}

知识点十五 静态

一、类和结构体外部的静态（全局静态）

不需要变量是全局变量时，就可以使用静态变量

//extern 表示使用了其他 .cpp文件的 var 名的变量
//类似，函数的声明，而实现在其他 .cpp文件（实际上函数只是隐式地使用了 extern 关键字）
extern int var = 5;

//static 在类和结构体外部的关键字，会使得其他所有翻译单元(如.cpp文件)都无法链接到这个变量，即使使用 extern 关键字都不行，类似类中的私有变量
static int var2 = 12;

//函数也一样
static void Function()
{

}

int main()
{

}

二、类和结构体中的静态（静态类成员）

在所有类的实例中，静态类成员都是同一个
因此，在某一个实例中改变这个静态类成员，他会反映到所有实例的静态类成员中

静态类成员函数无法使用普通的成员变量，只能使用静态成员变量
因为每个非静态成员方法，会获得当前类的一个实例作为参数，而静态类方法不会获得任何类的实例

class Entity
{
public:
    int x,y;

    void Print()
    {
        std::cout << x << ", " << y << std::endl;
    }
};

class StaticEntity
{
public:
    static int x, y;

    void Print()
    {
        std::cout << x << ", " << y << std::endl;
    }

    static void StaticPrint()
    {
        std::cout << x << ", " << y << std::endl;
    }
}

//静态的成员变量需要定义一下，才能在外部使用，这样链接器才能链接到
int StaticEntity::x;
int StaticEntity::y;

int main()
{
    //普通成员
    Entity e;
    e.x = 2;
    e.y = 3;
    //初始化
    Entity e1 = {5, 8};
    e.Print();      //打印2, 3
    e1.Print();     //打印5, 8

    StaticEntity se;
    se.x = 2;
    se.y = 3;
    Entity se1;
    //由于x，y是静态成员，第二次赋值时，实际上和上面那次是同样的变量
    se1.x = 5;
    se1.y = 8;
    se.Print();     //打印5, 8
    se1.Print();    //打印5, 8

    //可以这样使用静态成员
    StaticEntity::x = 10;
    StaticEntity::y = 12;
    StaticEntity::StaticPrint();
}
