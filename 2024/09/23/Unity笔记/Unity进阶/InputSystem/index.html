    知识点一：InputSystem导入

    InputSystem包导入

    PackageManager 中导入 Input System

    选择开启哪一种输入模式

    选择 InputSystem 和 老InputManager 的启用情况
    File > Build Setting > Player Setting > Other > Active Input Handing
    可以同时启用也可以只启用其中之一，每次重启后会重启Unity

    知识点三 Input Action类

    一、Input Action是什么？

    InputAction 是 InputSystem 帮助我们封装的输入动作类。
    它的主要作用是，不需要我们通过写代码的形式来处理输入
    而是直接在 Inspector 窗口编辑想要处理的输入类型
    当输入触发时，我们只需要把精力花在输入触发后的逻辑处理上

    我们在想用于处理输入动作的类中
    申明对应的 InputAction 类型的成员变量(需引用命名空间UnityEngine.InputSystem)

    二、参数

    查看 InputAction类.xmind

    三、InputAction的使用

    1.启用输入检测

    public InputAction move;
move.Enable();

    2.操作监听相关

    需要出入一个 InputAction.CallbackContext 类

    //开始操作
move.started += (context) =>{   };
//触发
move.performed += (context) =>{  };
//结束操作
move.canceled += (context) =>{  };

    3.关键参数（传入的参数） CallbackContext

    1. 当前状态
      callbackContext.phase //枚举
      未启用        Disabled
      等待        Waiting
      开始        Started
      触发        Performed
      结束        Canceled
    2. 动作行为信息
      callbackContext.action.name
    3. 控件信息
      callbackContext.control
    4. 获取值
      callbackContext.ReadValue();
    5. 持续时间
      callbackContext.duration
    6. 开始时间
      callbackContext.startTime
