    知识点一 自定义菜单栏拓展

    一、特殊文件夹Editor

    Editor文件夹是Unity中的特殊文件夹
    所有Unity中编辑器相关的脚本都需要放在其中
    使用了Unity编辑器相关命名空间(UnityEditor)的脚本最终都不会打包出去
    我们需要将这些脚本放置到Editor文件夹中，避免打包时报错

    二、在Unity菜单栏中添加自定义页签

    特性名：MenuItem
    命名空间：UnityEditor

    作用：
    在菜单栏点击该页签时，将执行静态函数中的逻辑

    注意：
    不用在意继承的对象
    必须使用反斜杠/

    //必须是静态方法
//通过 " / " 的方式添加子栏，必须有至少一个 " / "
[MenuItem("GameTool/Test")]
private static void Test()
{

}

    三、在Hierarchy窗口中添加自定义页签

    命名空间：UnityEditor
    特性：MenuItem

    用法：在静态函数前加上，并且在页签命名中放入GameObject路径
    [MenuItem(“GameObject/页签”)]

    注意：
    不用在意继承的对象
    必须使用反斜杠/
    菜单栏的GameObject也会出现对应的页签

    //必须是静态方法
[MenuItem("GameObject/Test")]
private static void Test()
{

}

    四、在Project窗口中添加自定义页签

    命名空间：UnityEditor
    特性：MenuItem

    用法：在静态函数前加上，并且在页签命名中放入Assets路径
    [MenuItem(“Assets/页签”)]

    注意：
    不用在意继承的对象
    必须使用反斜杠/
    菜单栏的Asset也会出现对应的页签

    //必须是静态方法
[MenuItem("Assets/Test")]
private static void Test()
{

}

    五、在菜单栏的Component菜单添加脚本

    命名空间：UnityEngine
    特性：AddCompontMenu

    用法：
    在想要通过Component菜单添加的脚本前加上
    [AddCompontMenu(“一级页签/二级页签”)]

    注意：
    脚本需继承MonoBehaviour对象
    必须使用反斜杠/
    最后一级选项的名字可以和脚本名不同，但建议一致

    //必须是静态方法
[MenuItem("SelfComponent/Test")]
public class SelfComponent : MonoBehaviour
{

}

    六、在Inspector为脚本右键添加菜单

    命名空间：UnityEditor
    特性：MenuItem

    用法：在静态函数前加上，并且在页签命名中放入CONTEXT路径
    [MenuItem(“CONTEXT/页签”)]

    注意：
    不用在意继承的对象
    必须使用反斜杠/

    //必须是静态方法
[MenuItem("CONTEXT/Test")]
private static void Test()
{

}

    七、加入快捷键

    单键
    路径 + 空格 + 下划线 + 按键

    //必须是静态方法
[MenuItem("CONTEXT/Test _F4")]
private static void Test()
{

}

    组合键
    下划线改为

    • %表示ctrl
    • #表示shift
    • &表示alt

    其他支持的按键
    LEFT、RIGHT：#LEFT，左shift
    UP、DOWN、F1、F12、HOME、END、PGUP、PGDN

    知识点二 自定义窗口拓展

    一、创建窗口类

    命名空间：UnityEditor
    类名：EditorWindow

    当我们想要为Unity拓展一个自定义窗口时
    我们只需要实现一个继承了EditorWindow的类即可
    并且在该类的OnGuI函数中编写面板控件相关逻辑

    二、显示窗口

    调用：
    EditorWindow.GetWindow

    该方法用很多重载
    Type或T：窗口类的类型
    utility：为 true 可创建浮动程序窗口，false 可创建正常窗口，所谓浮动窗口就是可以自由拖动自由改变大小
    title：窗口标题
    focus：是否为窗口提供焦点(如果已存在)。(如果 GetWindow 创建新窗口，则将始终获得焦点)
    desiredDockNextTo：窗口试图停靠到其上的 EditorWindow 类型的数组

    创建窗口对象
    之后调用对象的Show方法即可显示窗口

    public static void ShowWindow()
{
    TestClass tc = EditorWindow.GetWindow<TestClass>();
    tc.Show();
}

    三、窗口事件回调函数

    继承了EditorWindow的窗口类，自带一些事件回调函数
    当触发对应事件时会自动进入

    • OnHierarchyChange() 当场景中的层次架构(Hierarchy)发生变化时调用。 例如，当游戏对象被创建时触发
    • OnFocus() 当窗口获得焦点时调用。在这个方法中，可以执行一些在窗口中获得焦点时需要进行的操作
    • OnLostFocus() 当窗口失去焦点时调用。 通常这个方法中执行一些在窗口失去焦点时需要进行的清理工作
    • OnProjectChange() 当项目资源发生变化时调用。例如，当添加、删除项目中的文件时触发
    • OnInspectorUpdata() 在检视器(Inspector) 面板更新时调用。可以在这个方法中执行需要再检视器面板刷新时进行的逻辑
    • OnSelectionChange() 当选择的对象发生变化时调用。在这个方法中，你可以执行与所选对象相关的操作

    四、窗口中常用的生命周期函数

    • OnEnable() 当窗口被激活时调用，通常在创建时会调用一次。在这个方法中，你可以进行一些初始化工作，例如注册事件监听器或设置初始变量
    • OnGUI() 每帧都会调用此方法，用于绘制编辑器窗口的 GUI。在这个方法中，你可以使用 GUILayout 或 EditorGUILayout 等类创建界面元素
    • OnDestory() 当窗口被销毁时调用，通常在关闭编辑器或切换场景时触发。在这里进行清理工作，确保没有未被释放的资源
    • Updata() 在编辑器窗口每帧更新时调用，通常在这里执行一些需要在每帧进行的逻辑

    五、编辑器窗口类中的常用成员

    1.静态变量

    focusedWindow：当前已获得键盘焦点的 EditorWindow (只读)
    mouseOverWindow：当前在鼠标光标下的 EditorWindow (只读)

    2.静态函数

    CreateWindow：创建窗口，如果允许一个窗口有多个可以用该API创建窗口
    GetWindow：通过它我们可以创建一个窗口对象
    GetWindowWithRect：返回一个指定位置、大小的窗口
    HasOpenInstance：检测编辑器窗口是否打开

    3.成员变量

    titleContent：窗口标题名
    position：窗口位置大小信息
    wantsMouseEnterLeaveWindow：如果设置为true，则每当鼠标进入或离开窗口时，该窗口都会受到一次 OnGui调用

    4.成员函数

    Show：显示面板
    Repaint：重绘窗口
    Close：关闭窗口

    知识点三 EditorGUI

    一、EditorGUI是什么

    1.知识回顾

    GUILayout公共类
    一个GUI自动布局的公共类

    GUILayoutOption 布局选项
    控件固定宽高
    GUILayout.Width(300);
    GUILayout.Height(200);
    允许控件的最小宽高
    GUILayout.MinWidth(50);
    GUILayout.MinHeight(50);
    允许控件的最大宽高
    GUILayout.MaxWidth(100);
    GUILayout.MaxHeight(100);
    允许或禁止水平拓展
    GUILayout.ExpandWidth(true);
    GUILayout.ExpandHeight(false);

    GUILayout.Buttin("确定", GUILayout.Width(200));

    2.EditorGUI是什么

    EditorGUI 类似 GUI
    是一个主要用于绘制编辑器拓展 UI 的工具类
    它提供了一些 GUI 中没有的API
    主要是 编辑器功能中会用到的一些 特殊控件

    而 EditorGUILayout 类似于 GUILayout
    是一个带有自动布局功能的 EditorGUI 绘制工具类

    我们经常会将 EditorGUI 和 GUI 混合使用，来制作一些编辑器拓展功能
    但是由于更多时候我们会用到自动布局功能

    二、文本、层级和标签、颜色拾取 控件

    1.EditorGUILayout中的文本控件

    EditorGUILayout.LabelField("文本标题", "文本内容");
EditorGUILayout.LabelField("文本内容");

    2.EditorGUILayout中的层级、标签选择

    在自定义创建的编辑器窗口中，显示Tag和Layer相关的设置(类似游戏对象的)

    //层级Layer
int layer;
layer = EditorGUILayout.LayerField("层级选择", layer);
//标签Tag
string tag;
tag = EditorGUILayout.TagField("标签选择", tag);

    3.EditorGUILayout中的颜色获取

    color c = new color();
c = EditorGUILayout.ColorField(new GUIContent("标题"), c, 是否显示拾色器, 是否显示透明度通道, 是否支持HDR);

    三、枚举、选择、按下按钮 控件

    1.枚举选择控件

    public enum E_Type
{
    Big = 1,
    Small = 2,
    Max = 3,
    BigAndSmall = 1 | 2,
}
E_Type type = E_Type.Big;
//枚举选择
type = (E_Type)EditorGUILayout.EnumPopup("标题", type);

//多选枚举
//注意：多选枚举进行的是或运算，声明枚举时一定要注意其中的赋值，并且一定要有多种情况的搭配值
type = (E_Type)EditorGUILayout.EnumFlagsField("标题", type);

    2.整数选择控件

    string[] strs = {"选择一", "选择二", "选择三"};
int[] ints = {1,2,3};   //返回的是数组中的值
int a;
a = EditorGUILayout.IntPopup("标题", a, strs, ints);    //返回的是数组中的值

    3.按下就触发的按钮控件

    EditorGUILayout.DropdownButton( new GUIContent("按钮上的文字"), FocusType.Passive);

    FocusType枚举告诉UI系统能够获得键盘焦点，当用户按Tap键时，在控件之间进行切换
    Keyboard 该控件可接收键盘焦点
    Passive 该控件不可接收键盘焦点

    四、对象关联、各类型输入 控件

    1.对象关联控件

    对象变量 = EditorGUILayout.ObjectField(对象变量, typeof(对象类型), 是否允许关联场景上对象资源) as 对象类型;

    GameObject obj;
obj = EditorGUILayout.ObjectField(obj, typeof(GameObject), false) as GameObject;

    2.各类型输入

    int变量 = EditorGUILayout.IntField("int输入框", int变量);
long变量 = EditorGUILayout.LongField("long输入框", long变量);
float变量 = EditorGUILayout.FloatField("float输入框", float变量);
double变量 = EditorGUILayout.DoubleField("double输入框", int变量);

string变量 = EditorGUILayout.TextField("Text输入框", string变量);
Vector2变量 = EditorGUILayout.DoubleField("Vector2输入框", Vector2变量);
Vector3变量 = EditorGUILayout.DoubleField("Vector3输入框", Vector3变量);
Vector4变量 = EditorGUILayout.DoubleField("Vector4输入框", Vector4变量);

rect变量 = EditorGUILayout.DoubleField("rect输入框", rect变量);
bounds变量 = EditorGUILayout.DoubleField("bounds输入框", bounds变量);
boundsInt变量 = EditorGUILayout.DoubleField("boundsInt输入框", boundsInt变量);

//注意：EditorGUILayout 中还有一些Delayed开头的输入控件，这和普通输入控件的主要区别是：在用户按 Enter 键或将焦点从字段移开之前，返回值不会更改

    五、折叠、折叠组 控件

    1.折叠控件

    bool变量 = EditorGUILayout.Foldout(bool变量, “标题名”);

    bool b;
b = EditorGUILayout.Foldout(b, "折叠控件");

if(b)
{
    //要折叠的代码
}

    2.折叠组控件

    bool变量 = EditorGUILayout.BeginFoldoutHeaderGroup(bool变量, “标题名”);

    bool变量 = EditorGUILayout.EndFoldoutHeaderGroup();

    bool b;
b = EditorGUILayout.BeginFoldoutHeaderGroup(b, "折叠组控件");

if(b)
{
    //要折叠的代码
}

EditorGUILayout.EndFoldoutHeaderGroup();

    六、开关、开关组控件

    1.开关

    bool变量 = EditorGUILayout.Toggle(“普通开关”, bool变量);
    bool变量 = EditorGUILayout.ToggleLeft(“开关在左侧”, bool变量);

    bool b;
b = EditorGUILayout.Toggle("普通开关", b);

    2.开关组

    bool变量 = EditorGUILayout.BeginToggleGroup(“开关组”, bool变量);
    //包裹其他控件，控制这部分控件是否失活，或者激活
    EditorGUILayout.EndToggleGroup();
