    知识点一 GameEntry类

    一、简介

    GameEntry类是一个静态类，可以通过它获取到GF框架的其他的组件进行使用，比如FSM、Procedure等
    GameEntry在Start()中会调用两个方法：InitBuiltinComponents()和InitCustomComponent()，InitBuiltinComponents()中会对GF框架自带的组件进行初始化，而InitCustomComponent()中主要用于对自定义组件进行初始化

    二、如何自定义一个GameEntry类的组件

    1.继承GameFrameworkComponent

    声明一个类，继承GameFrameworkComponent，GameFrameworkComponent内部会自动将此类注册到GameEntry类中

    2.在GameEntryCustom文件中声明一个静态的自定义类

    在文件GameEntryCustom中（实际上还是GameEntry类）声明一个静态自定义属性，并且在InitCustomComponent()方法中使用 UnityGameFramework.Runtime.GameEntry.GetComponent<>()方法进行获取

    知识点二 流程 Procedure

    GF框架中的流程是基于FSM状态机实现的

    一、写一个自定义流程类

    继承 ProcedureBase

    二、改变流程API

    ChangeState<>();

    三、总流程

    1.主流程

    ProcedureLaunch(配置游戏基础信息) > ProcedureSplash(编辑器模式下，直接进入预加载流程；否则，检查版本，看是否需要热更新)

    2.是否编辑器模式下

    非编辑器模式下 > ProcedureCheckVersion(版本检测，热更新流程) > ProcedurePreload
    编辑器模式 > ProcedurePreload(开始加载游戏运行所必不可少的资源) > ProcedureChangeScene

    3.切换场景流程

    ProcedureChangeScene(切换场景)
    ProcedureMain(游戏主流程，从中实现了游戏逻辑)
    ProcedureMenu(游戏菜单流程)
    切换场景流程，会根据条件，切换游戏主流程和游戏菜单流程

    知识点四 数据表 DataTable

    一、简介

    将游戏数据以表格形式进行配置后，使用此组件加载这些数据表，数据表是可以自定义的

    二、为什么用数据表储存数据？

    当游戏制作到一定完成度，只需要对配置表进行配置，游戏就会自动实现其中的功能
    这样可以方便以后动态扩展与改变，容易进行调试

    三、数据表txt文件

    1.txt文件格式

    本身是一个excel形式存储的
    使用Excel打开txt文件进行修改
    修改完毕，保存时，点击另存为 > 文本文件(制表符分隔)(*.txt)
    若修改后出现乱码：打开txt文件 > 文件 > 另存为 > 编码 > UTF-16 LE

    2.新建数据表txt

    用Excel，按照规定进行填写并保存

    //自定义一个特殊类型数据，用于从Excel表生成数据结构类时，可以使用，比如Vector3
public sealed partial class DataTableProcessor
{
    private sealed class Vector3ListProcessor : GenericDataProcessor<List<Vector3>>
    {
    //是否是系统类型
        public override bool IsSystem
        {
            get
            {
                return false;
            }
        }
        //关键字
        public override string LanguageKeyword
        {
            get
            {
                return "List<Vector3>";
            }
        }
        //type类型名
        public override string[] GetTypeString()
        {
            return new string[]
            {
                "List<Vector3>",
                "System.Collections.Generic.List"
            }
        }
        public override List<Vector3> Parse(string value)
        {
            List<Vector3> temp = new List<Vector3>();
            if(value == "empty" || value == "")
            {
                return temp;
            }
            //分隔方式是自定义的
            string[] splitValues = value.Split(';');
            for(int i = 0; i < splitValues.Length; i++)
            {
                string[] splitValue = splitValues[i].Split(',');
                temp.Add(new Vector3(float.Parse(splitValue[0]), float.Parse(splitValue[1]), float.Parse(splitValue[2])));
            }
            return temp;
        }

        public override void WriteToStream(DataTableProcessor dataTableProcessor, BinaryWriter binaryWriter, string value)
        {
            List<Vector3> listVector3 = Parse(value);
            binaryWriter.Write((float)listVector3.Count);
            for(int i = 0; i < listVector3.Count; i++)
            {
                binaryWriter.Writer(listVector3[i].x);
                binaryWriter.Writer(listVector3[i].y);
                binaryWriter.Writer(listVector3[i].z);
            }
        }
    }
}

public static List<Vector3> ReadListVector3(this BinaryReader binaryReader)
{
    List<Vector3> temp = new List<Vector3>();

    float count = binaryReader.ReadSingle();
    for(int i = 0; i < count; i++)
    {
        temp.Add(new Vector3(binaryReader.ReadSingle(), binaryReader.ReadSingle(), binaryReader.ReadSingle()));
    }

    return temp;
}

    知识点五 事件系统 Event

    一、简介

    事件系统可以方便地订阅、抛出事件
    在游戏开发中，开发一些成就系统，比如杀死一个怪物，需要通知玩家的经验值增加等
    都可以使用事件系统来方便地获取信息

    二、GameEventArgs类

    GameEventArgs类，是一个事件信息类，可以继承它，创建自己的事件信息数据结构，用于在事件的回调中传输数据
    GF框架内部有自己准备好的一些GameEventArgs类

    三、订阅事件

    API： GameEntry.EventComponent.Subscribe(int id, EventHandler handler)
    参数一：事件id，不能重复订阅相同id且相同逻辑的事件，除非使用了取消订阅
    参数二：委托，public delegate void EventHandler(object sender, TEventArgs e); 其中sender就是发送者，e是EventArgs类
    注意：订阅时，并未真正调用委托，只是记录了事件的信息

    四、抛出事件

    API： GameEntry.EventComponent.Fire(object sender, GameEventArgs e)
    参数一：发送者
    参数二：GameEventArgs类的实例
    注意：
    此时才真正调用委托，其中的 GameEventArgs类的实例 在委托函数结束后，会被释放掉，可以在传入时，new 实例

    五、取消订阅事件

    API: GameEntry.EventComponent.Unsubscribe(int id, EventHandler handler)
    参数一：事件id，不能重复订阅相同id的事件，除非使用了取消订阅
    参数二：委托，public delegate void EventHandler(object sender, TEventArgs e); 其中sender就是发送者，e是EventArgs类
    此时，事件被取消订阅，该事件id又可以被用于订阅事件

    知识点七 引用池 Reference Pool

    一、简介

    即对象池，使用对象池，有助于减少Unity的GC，减少性能消耗
    与 Object Pool 作用类似，但 Object Pool 是主要是用于Unity中的物体

    二、几种计数标记

    Unused：未使用的，在引用池中，还未使用的对象
    Using：正在使用，从引用池中取出来，正在使用的对象
    Acquire：获取的，从引用池中获取的对象
    Release：释放的，使用完的对象，释放回引用池中
    Add：添加的，所有添加和添加过到引用池中的对象的数量
    Remove：移除的，已从引用池中移除的对象的数量，无法再得到

    三、添加引用

    API: ReferencePool.Add()
    向引用池中添加引用，但未使用

    四、获取引用

    API: ReferencePool.Acquire()
    从引用池中获取引用，若引用池中没有空闲(Unused)的对象，则直接创建一个对象

    五、释放引用

    API: ReferencePool.Release()
    释放一个对象回引用池中

    六、移除引用

    API: ReferencePool.Remove()
    从引用池中移除引用，引用池无法再次使用该对象
